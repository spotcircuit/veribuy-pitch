01
Weeks 1–4

Structured Ingestion

The foundation. Every photo, every signal, structured from the moment it hits the system.

Validate & Store

MIME checks, pHash dedup, size limits, malware scan. Raw → S3 with structured paths.

Extract Signals

EXIF, GPS, quality scores (blur, brightness, contrast), face/plate detection & auto-blur.

Normalize

Standardized JPEG at 2048px, 256px thumbnails, CLIP embeddings stored in pgvector.

Asset Graph

Link inspections to assets over time. This is where the compounding data moat starts.

02
Weeks 3–6

Training Data Infrastructure

Most teams skip this and regret it. I'd build outcome capture from day one.

  • Label Studio — Self-hosted annotation tool, not a custom UI. Focus effort on the adapter, not the interface.
  • 6-Dimension Taxonomy — Exterior paint, body panels, glass, wheels, interior, mechanical. Locked before first annotation.
  • Inter-Rater Tracking — Dual-annotate 20% of images. If MAE > 2.0 on any dimension, rewrite the guideline.
  • Outcome Capture — Loan approved? Claim denied? That signal flows back and becomes ground truth for model training.

The outcome feedback loop is what turns this from a scoring tool into a learning system.

03
Weeks 5–9

Hybrid Inference

Don't rip out the external API. Gradually replace it as the internal model proves itself.

Shadow

External API is primary. Internal model runs silently alongside. Log every divergence.

Canary

Route 10% to internal when confidence > 0.75. Monitor closely. Expand or pull back.

Graduate

Internal model becomes default for proven asset types. External API stays as fallback.

Base model: CLIP ViT-L/14 with custom regression head. Fine-tuned, not trained from scratch.

04
Weeks 8–12

Explainable Scoring

The output insurers and lenders see. Every score backed by evidence they can audit.

Decomposed Score

Weighted dimensions. Worst-case for structural damage. Composite 1–100 with tier labels.

Evidence Layer

Bounding boxes, attention heatmaps, contribution deltas. Show exactly what drove the score.

Report API

JSON for integrations, PDF for humans. Full audit trail: model version, route decision, confidence.

Explainability isn't a nice-to-have — it's the difference between a tool insurers adopt and one they reject.

Challenges

How I'd handle the hard parts

Noisy, Inconsistent Media

Quality signals (blur, exposure, occlusion) computed at ingest. Low-quality images get down-weighted in scoring, never silently ignored.

Incomplete Inspections

Coverage scoring per inspection. Report shows exactly which areas were/weren't photographed. Clients see the gaps.

Workflow Integration

Clean REST API with webhook callbacks. JSON reports for programmatic consumption. PDF for human review. No custom integration per client.

Improving Over Time

Outcomes (loan denied, claim paid) tagged back to original inspection. Materialized training view refreshes nightly. Model retrains at label thresholds.

Data Strategy

Getting to 500 labels fast

The model doesn't need to be good at first. It needs training infrastructure and labeled data flowing.

Retroactive Labeling

Take every existing prototype inspection through Label Studio with domain experts. 2 people, 2 days, 1,600 images labeled.

Synthetic Augmentation

5–10x variants per labeled image. Labels inherited. Training diversity without annotation cost.

Partner Outcomes

Process insurer backlog for free. Get outcome labels (loan approved/denied) in return. Worth 10x manual annotations.
